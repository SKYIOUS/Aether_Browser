# Aether Browser ‚Äî Architecture Context

## Engine Architecture

This is a **completely independent browser engine** written in Rust from scratch. It does NOT use Chromium, WebKit, or Gecko. All parsing, layout, and rendering is custom.

### Crate Layout (Cargo Workspace)

- **`aether-dom`** ‚Äî Pure DOM data structures (`Node`, `NodeType`, `ElementData`), plus the shared `StyledElement` type used across layout and render. Uses a flat `Vec` arena for DOM storage (not `Rc<RefCell>` tree pointers).
- **`aether-html`** ‚Äî HTML tokenizer and tree builder (takes raw strings, outputs `aether-dom` `Node` trees). Follows the HTML5 specification algorithm.
- **`aether-css`** ‚Äî The **Stratus** CSS engine: parser, cascade/specificity resolver, and `ComputedStyle` with all CSS properties. Codegen from `css-properties.json` generates the `CssPropertyName` enum. Property enums like `Display`, `Position`, `FlexDirection` use `strum` for automatic `FromStr`/`Display`.
- **`aether-caelum`** ‚Äî Block/flex layout engine embedded from the Taffy project. Layout coordinates use fractional pixels. Exposes `CaelumTree` + `Style` types.
- **`aether-layout`** ‚Äî Orchestrates the layout pipeline: `apply_caelum_layout()` integrates Caelum for block positioning, `layout_inline_run()` packs inline formatting context runs, `wrap_text()` does word-level line breaking, `apply_text_wrapping()` clips wrapped text to element width.
- **`aether-render`** ‚Äî Iced 0.13 canvas rendering: `PageCanvas` draws elements at Caelum-computed positions with background fills, borders, and text (using cosmic-text via `Shaping::Advanced` for font fallback).
- **`aether-browser`** (binary crate + glue) ‚Äî Entry point, `BrowserScreen` state machine, `fetch_page_content()` pipeline orchestrator, JS engine glue, networking, logging. Depends on all workspace crates.

### Non-Standard Choices

- **DOM Storage:** Flat `Vec<Node>` with `parent_index: Option<usize>`, *not* reference-counted tree pointers. Enables fast serialization and simple `slice` iteration.
- **Layout:** Caelum (Taffy fork) computes X positions; manual vertical flow with inline formatting context (IFC) line-box packing computes Y positions. Caelum's Y is overridden.
- **Text Layout:** Word-level wrapping via `wrap_text()` (character-count heuristic: `font_size * 0.58` per char), NOT a HarfBuzz-shaped engine. Lines are pre-computed before layout.
- **CSS Resolution:** Single-pass cascade via Stratus. Only inline `<style>` blocks and one `<link rel="stylesheet">` are processed (20 and 20 limits). External CSS capped at 100KB.
- **JS Engine:** rquickjs 0.11 wrapper. DOM is mirrored as a flat `JsBridge` behind `Arc<Mutex<>>`. Timers/events re-execute source strings (closures don't serialize). APIs: `document.*` (getElementById, createElement, createTextNode, querySelector, cookies, title), `window` (setTimeout, setInterval, fetch, XMLHttpRequest, localStorage, history shim), `console.*`, `element.style.*`.
- **Image Rendering:** Placeholder rectangles via Iced 0.13 `draw_image`. Actual pixel decoding via `image` crate. No CSS `object-fit` or `background-size` yet.
- **Networking:** Blocking ureq 3.3 (HTTP-only). 15s timeout, 60s HTTP cache TTL, 500KB HTML cap, 100KB CSS cap.

### Rendering Pipeline (fetch_page_content)

1. Fetch HTML (ureq, max 500KB)
2. Parse DOM (aether-html)
3. Extract `<style>` blocks and `<link rel="stylesheet">` URLs
4. Fetch external CSS (max 100KB)
5. Parse all CSS through Stratus (20 inline style blocks max, 20 external sheets max)
6. Extract `<script>` tags, execute through JsBridge
7. `extract_elements()` ‚Äî walks DOM, computes Stratus styles ‚Üí `Vec<StyledElement>` (depth 30, max 300 elements)
8. Fetch `<img>` bytes (main thread blocking)
9. `apply_caelum_layout()` ‚Äî Caelum tree + manual vertical flow (IFC line-box packing)
10. `apply_text_wrapping()` ‚Äî word-break wrapping using element widths

### Key Constraints

- `wrap_text()` uses `font_size * CHAR_W_SCALE` (0.58) as character width ‚Äî monospace approximation. Non-Latin widths are inaccurate.
- `Shaping::Advanced` enables cosmic-text font fallback for Cyrillic/CJK ‚Äî but only when the system has matching fonts installed (Noto Sans, etc.)
- Max 300 elements, depth 30, text 1000 chars per element
- `overflow-checks = false` in dev profile ‚Äî cosmic-text u16 overflow in debug
- Border widths per-element: `[top, right, bottom, left]`, drawn as thin rectangles
- Click hit-testing uses element bounding box, exact text extents not available
